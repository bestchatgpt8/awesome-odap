# Contributing to Awesome ODAP

Thanks for helping map the Open Data Agent Platform landscape.

## Inclusion Criteria

A project, paper, product, or standard can be included if it supports at least
one ODAP layer:

- universal data access
- open table formats
- semantic layers or metric governance
- natural-language analytics
- text-to-SQL
- multi-agent orchestration
- persistent memory
- observability, lineage, quality, audit, policy, or cost controls

## Style

- Use neutral descriptions.
- Prefer official websites, docs, or repositories.
- Avoid superlatives unless the project itself uses them and the wording is
  attributed.
- Include competitors and adjacent tools. The list is meant to map the field,
  not promote a single vendor.
- Keep entries alphabetized inside each section when possible.

## Entry Template

```markdown
- [Project Name](https://example.com/) - Neutral one-sentence description.
```

## Not Accepted

- Pure affiliate links.
- Unverifiable claims.
- Duplicate entries.
- Products unrelated to data, analytics, agents, semantics, or governance.
